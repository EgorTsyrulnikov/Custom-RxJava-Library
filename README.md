## Описание проекта

Цель данной курсовой работы — реализовать систему реактивных потоков данных, используя паттерн «Наблюдатель» (Observer). 
Проект содержит реализацию ключевых компонентов оригинальной библиотеки **RxJava** с поддержкой асинхронного выполнения, обработки ошибок и применения операторов-преобразователей (Operators).

### Основные возможности
1.  **Базовые компоненты потока**: `Observable`, `Observer`, `Emitter`, `Disposable`.
2.  **Операторы трансформации**: `map`, `filter`, `flatMap`.
3.  **Асинхронность и многопоточность**: Собственные реализации интерфейса `Scheduler` (`IOThreadScheduler`, `ComputationScheduler`, `SingleThreadScheduler`), а также операторы переключения контекста потоков `subscribeOn` и `observeOn`.
4.  **Безопасность**: Защита от состояния гонки (race conditions) при терминальных событиях (`onComplete`, `onError`).

## Запуск проекта и тестов

Проект использует систему сборки **Maven** и написан на **Java 11+**.

### 1. Демонстрационный пример
Чтобы запустить `Main.java` и увидеть наглядную работу многопоточности в консоли, выполните:
```bash
mvn clean compile exec:java -Dexec.mainClass="com.customrx.Main"
```
*(Пример выведет в консоль сгенерированные данные с указанием потоков пулов IO и Computation, в которых они обрабатывались).*

### 2. Запуск Unit-тестов
Проект покрыт автоматическими тестами (JUnit 5), которые проверяют работу операторов, обработку ошибок, корректность отмены подписок (`dispose`) и синхронизацию переключения пулов потоков.
Для запуска тестов выполните:
```bash
mvn clean test
```
